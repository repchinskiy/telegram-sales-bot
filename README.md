# Telegram Sales Bot

Telegram-бот на Python для первичной коммуникации с клиентом, FAQ-ответов, маршрутизации запросов и передачи диалога оператору.

## Что умеет

- Приветствует пользователя и показывает стартовое меню.
- Обрабатывает типовые вопросы через заранее заданные ответы.
- Маршрутизирует входящие сообщения: FAQ, сбор контактов, перевод на оператора.
- Передает диалог в операторский чат Telegram.
- Позволяет оператору взять заявку в работу и отвечать клиенту через reply.
- Поддерживает режимы `bot`, `handoff_requested`, `human`, `closed`.

## Как это работает

1. Пользователь пишет боту в Telegram.
2. Telegram передает update боту через Bot API.
3. `aiogram` получает update, прогоняет его через router/filters/handlers и вызывает нужную логику.
4. Внутренние сервисы решают, что делать дальше: ответить шаблоном, собрать контакт или подключить оператора.
5. Если нужен оператор, бот отправляет заявку в операторский чат, а ответы оператора пересылает обратно клиенту.

## Основные компоненты

### Handlers

Обрабатывают входящие события Telegram: сообщения, callback-кнопки, команды.

### Services

Содержат прикладную логику:
- `routing_service` — определяет, что делать с сообщением.
- `handoff_service` — передает диалог оператору и пересылает сообщения.
- `dialogs` — хранит шаблонные тексты.
- `conversations` — управляет состоянием диалога.

### Bridge

Хранит соответствие между сообщениями в операторском чате и `client_user_id`, чтобы оператор мог отвечать клиенту через reply.

## Сценарий работы с оператором

1. Клиент нажимает кнопку связи с оператором или пишет сообщение, которое попадает под handoff.
2. Бот отправляет карточку заявки в операторский чат.
3. Оператор нажимает `Взяти в роботу`.
4. Оператор отвечает reply на карточку или на следующее сообщение клиента.
5. Бот доставляет ответ обратно клиенту.
6. После завершения оператор закрывает обращение.

## Запуск локально

### 1. Создать виртуальное окружение

```bash
python -m venv .venv
```

### 2. Активировать окружение

Windows:

```bash
.venv\Scripts\activate
```

Linux/macOS:

```bash
source .venv/bin/activate
```

### 3. Установить зависимости

```bash
pip install -r requirements.txt
```

### 4. Настроить переменные окружения

Создать файл `.env`:

```env
BOT_TOKEN=your_telegram_bot_token
OPERATOR_CHAT_ID=-1001234567890
```

### 5. Запустить бота

```bash
python main.py
```

## Запуск в фоне на Windows

Для постоянной работы на Windows удобно запускать бота как службу через NSSM.

Пример `run_bot.bat`:

```bat
@echo off
cd /d C:\projects\sales-bot
call .venv\Scripts\activate.bat
python main.py >> logs\bot.log 2>&1
```

Далее сервис можно создать через:

```bat
nssm install SalesBot
```

И указать:
- Path: `C:\Windows\System32\cmd.exe`
- Startup directory: `C:\projects\sales-bot`
- Arguments: `/c C:\projects\sales-bot\run_bot.bat`

## MVP-ограничения

- Состояние может храниться в памяти процесса.
- После рестарта процесса временные mapping-данные могут теряться.
- FAQ и routing построены на простых правилах и ключевых словах.
- Для production-версии желательно добавить постоянное хранилище, логирование и админские инструменты.
